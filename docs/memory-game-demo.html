📝 How It Works English

Reactive State Declaration

fn memory_game() uses ui:
    # Reactive variables auto-update UI
    observe var cards = [1,2,3,4,1,2,3,4]
    observe var revealed = [false]*8
    observe var matched = [false]*8
    observe var matches_found = 0

Each observe var declaration creates a reactive binding. Changes automatically trigger UI updates.

Event Handlers with Async Logic

async def handle_card_click(index):
    if is_checking or matched[index]:
        return

    revealed[index] = true
    # ... determine if match ...

    await asyncio.sleep(1.0)
    # Check result or reset
    first_pick = -1  # Updates UI instantly

Async logic with await suspends execution. Variable changes trigger reactive updates.

Reactive Rendering

render:
    for i in range(8):
        button class="card"
            class:revealed=(revealed[i])
            class:matched=(matched[i])
            onclick=handle_card_click(i):
            if matched[i]: "✓"
            else if revealed[i]: str(cards[i])
            else: "?"

The render: block automatically re-executes when observed variables change.

🎯 What This Demonstrates:
  • observe var for reactive state
  • on_change handlers for side effects
  • Async/await within reactive logic
  • Event binding (onclick)
  • Conditional rendering
  • Loop rendering over state
  • CSS class bindings

🌍 Polyglot Implementation French

The same game logic, written in French syntax. Identical functionality, different language:

French Version (Exact Same Logic)

déf jeu_memoire() utilise ui:
    # État du jeu - paires de cartes
    observer var cartes = [1,2,3,4,1,2,3,4]
    observer var revelees = [faux]*8
    observer var correspondantes = [faux]*8

    asynchrone déf gerer_clic_carte(indice):
        si en_verification ou correspondantes[indice]:
            retour

        revelees[indice] = vrai
        # ... identical game logic ...
💡 Key Insight: The French version uses completely different keywords:
  • déf (French) vs fn (English)
  • observer var (French) vs observe var (English)
  • asynchrone (French) vs async (English)
  • si (French) vs if (English)

Yet both compile to identical WASM and behave identically. This proves polyglot semantics.

🚀 Why Reactive UI Matters

No Framework Overhead

Game logic is pure language syntax. No React, Vue, or Angular needed. Less JavaScript, more productive.

Automatic Updates

Change a variable, UI updates instantly. No setState(), no hooks, no re-render complexity.

Polyglot-First

Write UI logic in your language of choice. English, French, Spanish, Japanese, Lisp — all work identically.

WASM-Ready

Compiles to WebAssembly. Runs with native performance. Perfect for games, dashboards, real-time apps.

Async-Native

async/await works seamlessly within reactive blocks. Sleep, delay, animate with language support.

Learnable

Simple model: state + reactions = UI. No hidden magic. Understanding one reactive app = understanding all.
